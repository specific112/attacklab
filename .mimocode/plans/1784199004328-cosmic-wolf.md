# Plan: Fix "Unknown" devices + Bolder dashboard text

## Issue 1: Devices showing "Unknown"

The user-agent parser only recognizes 5 browsers (Chrome, Firefox, Safari, Edge, Opera). Any other browser (Samsung Internet, Brave, Yandex, UC Browser, QQ Browser, etc.) falls through to "Unknown". Also, some sessions may have empty/null user-agent strings.

### Fix — `lib/user-agent.ts` and inline `parseUA` in `app/admin/users/page.tsx`

Expand browser detection to cover:
- Samsung Internet (`SamsungBrowser`)
- Brave (`Brave`)
- Yandex (`YaBrowser`)
- UC Browser (`UCBrowser` or `UCWEB`)
- QQ Browser (`QQBrowser`)
- Vivaldi (`Vivaldi`)
- Opera Mini (`Opera Mini`)
- Android WebView (`wv` or `; Android` without `Chrome`)
- Bot/Crawler detection as fallback

Also improve OS detection:
- Add `Windows NT 6.0` (Vista), `Windows NT 5.1` (XP) as fallback "Windows"
- Add `HarmonyOS` detection
- Add `Windows Phone` detection

Add a `fullUserAgent` passthrough: if nothing matches, return the first 60 chars of the raw UA string so it's not just "Unknown".
